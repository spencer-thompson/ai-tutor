Project Overview

What is the AI Tutor?

  • A generative AI-powered learning assistant for Utah Valley University (UVU) students.
  • Provides a personalized, interactive learning experience by integrating with the Canvas LMS.
  • The goal is to provide UVU students with a powerful and accessible tool to enhance their learning and academic success.
  • The project was started in early 2024 in discussion with the UVU Tech Management Department.

Project Timeline & Current Sprint

Key Requirements (Functional)

  • F1.1: Authentication: Users must log in via UVU credentials using Canvas OAuth2.
  • F1.2: API Security: The system must use JWT for authenticating all API requests.
  • F1.4: Admin Dashboard: Must display real-time analytics (total users, active users, messages sent).
  • F2.3: Real-time Chat: The system shall stream messages to the client.
  • F3.3: "Smart Chat": Must inject context from the user's Canvas data into the LLM prompt.
  • F3.5: AI Tools: The AI must be able to use a web scraper and a sandboxed Python interpreter.
  • F3.6: Moderation: All input and responses must pass through a content moderation filter.

Key Requirements (Non-Functional)

  • NF1.1: Encryption (In Transit): All network communication must use TLS 1.2 or higher.
  • NF1.4: Encryption (At Rest): All user data in the database must be encrypted (AES-256).
  • NF1.2: Web Security: Must be protected against OWASP Top 10 vulnerabilities (XSS, NoSQL Injection).
  • NF2.2: Performance: First token of AI response delivered in < 2 seconds (average).
  • NF4.1: Scalability: The system must be horizontally scalable.
  • NF5.1: Reliability: The system shall have 99.9% availability (uptime).
  • NF6.3: Maintainability: New code must have a minimum of 80% unit test coverage.

Testing Strategy

A multi-layered approach to validate the application:

Unit Testing (Isolation)

Tools: `pytest` (Backend), `Vitest` & `Svelte Testing Library` (Frontend).

Integration Testing (Modules Together)

Tools: `pytest`, `TestClient` for FastAPI (e.g., API endpoint writing to DB).

End-to-End (E2E) Testing (User Scenarios)

Tools: `Playwright` (e.g., full login-to-chat flow).

Security Testing (Vulnerabilities)

Methods: Dependency scanning, manual penetration testing (JWT, injection).

Quality Assurance & Metrics

Preventive QA

  • Static Code Analysis: `ruff` & `bandit`.
  • Code Reviews: All new code reviewed.
  • Coding Standards: Enforcing PEP 8.

Key Metrics

  • Test Coverage: Aim for 80% (backend).
  • Cyclomatic Complexity: Keep all functions < 10.
  • MTTR (Repair Time): < 24 hours for critical bugs.
  • MTBF (Failure Time): > 1000 hours.

User Feedback Analysis: The Good

Users are generally positive about the Canvas integration:

"I was surprised at how well it was able to access the information in Canvas. I was able to ask it some very specific questions about grades, and it had the answers."

"I asked what assignments were due this week and it pulled up all the assignments... They were all correct..."

"I think that it is super helpful that it connects to my canvas courses."

User Feedback Analysis: Areas for Improvement

Key issues identified related to stability, accuracy, and scope:

Stability:

"half of the time it crashed on me showing me an error sign..."

Accuracy:

"...it mixed up my classes, telling me that the classes I have on Monday... I have it on Tuesday..."

Complex Queries / Scope:

"When I asked if TechTip assignments were required, it said all assignments were required, which could be improved with syllabus access."

Action Plan & Next Steps

Based on user feedback, the following actions are proposed:

1. Improve Accuracy and Stability

  • Prioritize fixing bugs that cause crashes and incorrect information.
  • Enhance Canvas data retrieval for more accurate data.

2. Implement High-Impact Features

  • Highest Priority: Develop a feature to access and understand the course syllabus.
  • Create a better onboarding experience for new users.

3. Expand Knowledge Base

  • Explore integration with other school systems (events, announcements).
  • Investigate a map feature for course locations.

Core Technology Stack

Frontend: Svelte

Backend: Python with FastAPI

Database: MongoDB

Authentication: JWT & Canvas OAuth2

Containerization: Docker

Deployment: Hetzner VPS

Why MongoDB? Chosen because the Canvas API data is "messy" and unstructured, and MongoDB does not require a fixed schema.

Key Features

  • "Smart Chat" Functionality:
    • Securely accesses Canvas data (courses, assignments, grades).
    • Provides context-aware and personalized assistance.
  • Rich Text Interface:
    • Supports GitHub-flavored markdown.
    • Renders mathematical equations (LaTeX).
    • Syntax highlighting for code (Python, JS, etc.).
  • AI Tools:
    • Web Scraper: Reads text content from a URL.
    • Python Interpreter: Executes sandboxed code.
  • Personalization: Users can set a custom bio to influence the AI's tone.
